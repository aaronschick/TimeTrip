# CSV Template for Timeline Events

This CSV template matches the database schema for the TimeTrip application.

## Required Fields

- **id** (String, 100 chars max): Unique identifier for the event
- **title** (String, 500 chars max): Event title/name
- **category** (String, 100 chars max): Event category (e.g., "era", "civilization", "empire", "war", "religion", "biblical", "migration")
- **start_year** (BigInteger): Start year of the event (negative for BCE, e.g., -5000)
- **end_year** (BigInteger): End year of the event (can be same as start_year for instantaneous events)

## Optional Fields

- **continent** (String, 100 chars max): Geographic continent (default: "Global")
- **description** (Text): Detailed description of the event
- **start_date** (Date): Precise start date (format: YYYY-MM-DD, e.g., "2023-01-15")
- **end_date** (Date): Precise end date (format: YYYY-MM-DD)
- **lat** (Float): Latitude (-90 to 90)
- **lon** (Float): Longitude (-180 to 180)
- **location_label** (String, 500 chars max): Human-readable location name (e.g., "Rome, Italy")
- **geometry** (Text/GeoJSON): GeoJSON geometry string (e.g., `{"type": "Point", "coordinates": [lon, lat]}`)
- **location_confidence** (String): One of "exact", "approx", or "disputed" (default: "exact")

## Notes

- **created_at** and **updated_at** are auto-generated by the database and should not be included in the CSV
- Empty fields can be left blank in the CSV
- For GeoJSON geometry, use double quotes and escape inner quotes (e.g., `"{""type"": ""Point"", ""coordinates"": [lon, lat]}"`)
- Date fields should be in ISO format (YYYY-MM-DD)
- Years can be negative for BCE dates (e.g., -5000 for 5000 BCE)
- If end_year is not provided, it will default to start_year
